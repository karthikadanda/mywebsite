## Step-by-Step GitHub Pages Setup (Recommended)

### Prerequisites
- GitHub account (free)
- Your website files ready

### Steps:

#### 1. Create GitHub Repository
1. Go to https://github.com
2. Click "New repository"
3. Repository name: `xprint-solutions-website`
4. Description: "XPRINT Solutions - Ameerpet Printing Business Website"
5. Make it **Public**
6. Click "Create repository"

#### 2. Upload Files
1. Click "Add file" > "Upload files"
2. Drag and drop all your website files:
   - `index.html`
   - `script.js`
   - `README.md`
   - `sw.js`
   - `robots.txt`
   - `sitemap.xml`
3. Commit changes

#### 3. Enable GitHub Pages
1. Go to repository "Settings"
2. Scroll to "Pages" section
3. Source: Deploy from a branch
4. Branch: `main` > `/ (root)`
5. Click "Save"

#### 4. Get Your Live URL
After 2-5 minutes, your site will be live at:
`https://[your-username].github.io/xprint-solutions-website/`

## Custom Domain Setup (Optional)

### Option A: Free Subdomain
- Use GitHub Pages URL (free)
- Format: `username.github.io/xprint-solutions-website`

### Option B: Custom Domain (.in domain ~$300/year)
1. Buy domain from GoDaddy, Namecheap, etc.
2. Update DNS settings
3. Configure in GitHub Pages settings

### Option C: Free Domain Services
- Freenom (.tk, .ml, .ga) - limited reliability
- EU.org - free but manual approval
